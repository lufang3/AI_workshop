📥 실습 자료 다운로드

아래 버튼을 누르면 파일이 바로 내려받아집니다. (파일이 HTML 안에 포함되어 있어 인터넷 없이도 동작합니다.)

🧩 실습 프롬프트 모음 (Google AI Studio Build용)

각 프롬프트를 그대로 복사해서 Google AI Studio의 **Build** 탭 입력창에 붙여넣으세요. 결과가 의도와 다르면 "OO를 △△로 바꿔줘"라고 이어서 요청해 다듬으면 됩니다. (대화하며 고쳐가는 것이 바이브코딩의 핵심)

실습 1 — PPT 생성기

발표 개요(제목과 항목)를 입력하면 컨설팅 자료처럼 깔끔한 PPT를 자동으로 만들어주는 웹앱을 만들어줘. 항목 개수에 따라 카드·매트릭스 등 레이아웃이 자동으로 바뀌고, 숫자는 강조되며, 완성된 .pptx 파일을 다운로드할 수 있게 해줘.

실습 2 — 마감 관리 대시보드

업무명·담당자·마감일·상태를 입력해 관리하는 대시보드 웹앱을 만들어줘. 마감일까지 남은 날짜(D-Day)를 자동 계산해 임박한 건은 색으로 강조하고, 새로고침해도 데이터가 유지되며 CSV로 내보낼 수 있게 해줘.

실습 3 — 데이터 분석·보고문 생성기

엑셀 데이터를 붙여넣거나 업로드하면 자동으로 분석해주는 웹앱을 만들어줘. 항목별 합계·평균·최고·최저를 요약하고 차트로 보여주며, 분석 내용을 보고용 문장으로 자동 작성하고 결과를 엑셀로 내려받을 수 있게 해줘.

실습 4 — 월간 보고서 생성기

이번 달 업무 목록과 이슈·다음 달 계획을 입력하면 월간 업무 보고서를 자동으로 만들어주는 웹앱을 만들어줘. 완료율을 자동 계산해 보여주고, 완성된 보고서를 Word와 PPT 파일로 다운로드할 수 있게 해줘.

실습 5 — 회의록 정리 도구

회의 내용을 붙여넣으면 자동으로 안건·결정사항·할 일(담당자 포함)로 정리해주는 웹앱을 만들어줘. 정리된 회의록을 복사하거나 Word 파일로 내려받을 수 있게 해줘.

실습 6 — 이메일·공지문 작성 도우미

상황과 핵심 내용을 입력하면 정중한 업무용 이메일이나 사내 공지문을 자동으로 작성해주는 웹앱을 만들어줘. 정중함 정도(격식/캐주얼)를 선택할 수 있고 결과를 복사할 수 있게 해줘.

실습 7 — 텍스트 비교·교정 도구

두 개의 글을 나란히 붙여넣으면 달라진 부분을 표시해주고, 맞춤법과 어색한 표현을 다듬어주는 웹앱을 만들어줘. 교정된 결과를 복사할 수 있게 해줘.

🔎 IATF 16949 심사보고서 검증 프로그램 프롬프트

**사용법** 1. Google AI Studio의 **Build** 탭에서, 먼저 검증 기준 md 파일을 **첨부**합니다. - `IATF_심사보고서_주요_검증반려사항_및_보고서_작성지침_20251130_R0_.md` - (필요 시) `IATF_16949_심사보고서_작성_가이드_20260601_R0_최종.md` 2. 아래 "프롬프트" 박스를 통째로 복사해 함께 입력합니다. 3. 생성 후 결과가 의도와 다르면 "OO를 △△로 바꿔줘"로 이어서 다듬습니다.

검증 프로그램 생성 프롬프트 (그대로 복사)

첨부한 md 파일은 IATF 16949 심사보고서 작성 지침(검증 기준)이다. 이 첨부 파일을 검증 기준으로 삼아, 심사원이 작성한 심사보고서가 지침대로 작성되었는지 자동으로 검증해주는 React 웹앱을 만들어줘. 모든 UI는 한국어. Gemini 모델을 호출해 보고서 내용을 분석하고 지침 충족 여부를 판단한다.

[검증 기준 — 첨부 파일에서 추출]
- 첨부한 지침 md 파일의 각 항목(No.1 ~ 마지막 항목)을 검증 기준으로 사용한다.
- 각 항목의 "검증 반려사항"과 "보고서 작성 지침" 내용을 기준으로 보고서를 점검한다.
- 검증 기준 항목은 앱 안에 데이터로 내장하되, 사용자가 화면에서 확인·수정할 수 있게 한다. (지침 개정 대비)

[목적]
심사원이 작성한 심사보고서(텍스트 또는 Word 문서)를 붙여넣거나 업로드하면, 첨부 지침의 모든 항목에 대해 지침대로 작성되었는지 점검하고, 항목별로 적합/부적합/확인필요를 판정한 뒤 근거와 개선안을 제시한다.

[화면 구성]
- 헤더: "IATF 16949 심사보고서 검증 도구"
- 2단 구성: 좌측 입력 / 우측 검증 결과
- 상단에 [검증 시작] 버튼과 전체 결과 요약(적합 N건 / 부적합 N건 / 확인필요 N건)

[좌측 입력]
- 보고서 입력 방법 탭 2개:
  1. "텍스트 붙여넣기": 큰 textarea(최소 400px)
  2. "파일 업로드": .txt, .md, .docx 파일 업로드
     - .docx 파일은 브라우저에서 텍스트를 추출해 사용한다. (예: mammoth.js 등 클라이언트 라이브러리로 본문 텍스트 변환, 표 내용도 최대한 살림)
     - 업로드 후 추출된 텍스트를 미리보기로 보여주고, 텍스트 붙여넣기 영역에도 채워 사용자가 확인·수정할 수 있게 한다.
     - 지원하지 않는 형식이거나 텍스트 추출에 실패하면 한국어 오류 메시지를 표시한다.
- 입력 안내문: "프로세스맵, 요구사항 매트릭스, 준비성평가서, 심사세부일정표, CARA 보고서 등 보고서 본문을 붙여넣거나 .txt/.md/.docx 파일로 업로드하세요."

[우측 검증 결과 — 지침 항목별 카드]
각 검증 항목을 카드로 표시. 카드마다:
- 항목 번호와 제목 (첨부 지침의 항목명 그대로)
- 판정 뱃지: 적합(초록) / 부적합(빨강) / 확인필요(노랑)
- 판정 근거: 보고서에서 찾은 내용 또는 누락된 부분을 인용·요약
- 개선 제안: 부적합·확인필요인 경우, 지침에 맞게 보완할 방법을 한국어로 제시
- 적합 항목은 접어두고, 부적합·확인필요 항목을 위로 정렬

[검증 로직 — Gemini 호출]
- [검증 시작]을 누르면, 보고서 전문 + 첨부 지침의 검증 기준을 Gemini에 보내 각 항목을 판정하게 한다.
- 응답은 JSON으로 받아 카드에 매핑한다. 각 항목: { 번호, 제목, 판정(적합/부적합/확인필요), 근거, 개선제안 }
- 지침의 모든 항목을 누락 없이 전부 판정하도록 지시한다.
- 보고서 내용만으로 판단이 불충분하면 "확인필요"로 분류하고 무엇을 확인해야 하는지 적는다.

[문서 간 정합성 별도 점검 — 결과 하단]
보고서 안 여러 정보의 일치 여부를 교차 점검해 별도 표로 표시:
- 고객정보 (고객목록 / CARA 보고서 / 심사세부일정표 간)
- 근무시간 (CARA 보고서 / 심사세부일정표 / 인증범위 간)
- 외주처리 프로세스 종류 (프로세스맵 / CARA / 심사세부일정표 간)
- 원격지원기능 (프로세스맵 / CARA 보고서 간)
불일치 발견 시 빨간색으로 강조하고 어떤 문서끼리 다른지 명시한다.

[결과 내보내기]
- [검증 결과 다운로드] 버튼 → 항목별 판정·근거·개선제안을 정리한 보고서를 텍스트 또는 Word(.docx)로 다운로드
- 파일명: IATF심사보고서_검증결과_(오늘날짜)

[디자인]
- 강조 색: 진한 파랑 (공식 문서 톤)
- 적합 초록 / 부적합 빨강 / 확인필요 노랑으로 일관된 색 체계
- 카드형 레이아웃, 한국어 UI, 검증 중 로딩 표시

[기술 요건]
- React + TypeScript 단일 페이지 앱
- Gemini API 호출로 보고서 분석 (AI Studio 내장 연동 사용)
- 검증 기준은 첨부 md에서 가져와 내장하고, 화면에서 수정 가능하게
- .docx 파일 파싱을 위한 클라이언트 라이브러리(예: mammoth.js) 사용, 업로드 파일은 서버 전송 없이 브라우저에서 처리
- 입력·추출·다운로드 전 과정에서 한글이 깨지지 않도록 UTF-8로 처리

강사·운영 메모

  • 첨부 파일을 검증 기준으로 쓰는 구조의 장점: 지침이 개정되면 새 md만 첨부해 다시 생성하면 되므로, 프롬프트를 고칠 필요가 없습니다. 검증 항목 수가 바뀌어도 자동 반영됩니다.
  • 이 앱은 "참고용 1차 점검 도구"입니다. 최종 검증반려 여부는 인증기획센터의 공식 판단을 따라야 하며, 앱 판정을 그대로 신뢰하지 않도록 안내가 필요합니다.
  • 보고서 원본이 Word(.docx)인 경우 파일을 그대로 업로드하면 브라우저에서 본문 텍스트가 자동 추출됩니다. PDF/한글(hwp)인 경우에는 본문을 텍스트로 변환해 붙여넣는 흐름이 안정적입니다. (PDF 직접 파싱은 한글 추출 오류 위험)
  • 민감정보 주의: 실제 조직명·고객사·심사 데이터가 들어가므로, 무료 등급 Gemini는 입력이 모델 개선에 쓰일 수 있습니다. 실데이터 검증은 유료 API 키 또는 데이터 학습 비활용 설정이 가능한 환경에서 사용하도록 안내하세요.
